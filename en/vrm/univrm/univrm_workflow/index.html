    Workflow

    Prepare a 3D model that can be handled as “Humanoid” in Unity

    Prepare a humanoid model that can be imported into Unity such as FBX format. Navigate to model’sImport Settings -> Rig -> Animation Typeand selectHumanoidafter the model is imported.
    SelectHumanoid

    Correct bone assignment with rig configuration

    When the model file is imported, bone assignments done by auto-recognition may be different from expectations (see image below):

    • Jaw is assigned to mouth
    • Eye is assigned to eye highlight
    Since the eyebone (and jawbone) auto-recognition on “3D chan” is incorrect, we fix it by assigning the right component manually

    Export from menu
    If the 3D model data with setup-completed humanoid inHierarchyis selected, the options inVRMwill become available
    If the 3D model data with setup-completed humanoid inHierarchyis selected, the options inVRMwill become available
    Check the boxes and clickExportbutton

    Force T Pose

    Force the model pose to become T-Pose before removing rotation / scale.

    Change to T Pose manually

    You can manually change the model pose toT Poseinstead of changing it automatically. Make the model T-Pose manually in advance, navigate tomenu -> VRM -> export humanoidand uncheckForce T Pose.

    Pose Freeze

    Whether removal processing of rotation / scale is performed. It is the process of normalizing model for conforming to the VRM rules. Please make sure to check the boxes at the first time of use. By passing through this process beforehand, various components can work correctly.

    Please import vrm file into Assets folder
